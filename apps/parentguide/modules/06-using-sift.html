06 Using SIFT's tools

A real walkthrough, not a features list. Where each piece actually stands right now, honestly — some of this is further along than the rest.

Step one

Loading AdSentinel

AdSentinel isn't on the Chrome Web Store yet — right now it's loaded as an unpacked extension, which sounds more technical than it is: download the repository, open chrome://extensions, turn on Developer mode top right, click Load unpacked, and select the apps/adsentinel folder. Full steps are in that app's own README if any of that's unfamiliar.

Manual scan

Scanning a page

Click the SIFT icon in your toolbar, then Scan this page. Every flagged ad shows up with its type — dark pattern, age-restricted category, unverified network, no accessible text — and a plain-English reason underneath it, not just a label. Zero flags doesn't mean the page is clean; it means nothing text-based tripped a rule (Module 01 covers this limitation, worth re-reading if you skipped it).

Auto-watch

The automatic toast on YouTube

On YouTube specifically, AdSentinel also watches for video ads without you clicking anything — a small notice slides in from the corner the moment one starts playing, evaluated the same way a manual scan would be. This only runs on YouTube, Facebook, Instagram, and TikTok — the four domains AdSentinel explicitly asks permission for, and nowhere else.

Sharing

Export, and Share to AdDashboard

After a scan with flags, two separate things become available. Export saves the full result — including ad text — to a file on your own machine; nothing about that leaves your browser. Share to AdDashboard is a second, separate opt-in button that sends a much narrower slice — which platform, which flag types, nothing else — to the public aggregate statistics covered in the next module's cousin, AdDashboard.

Why two different buttons

Export is for you. Share is for everyone else, and deliberately can't carry your data with it — see packages/schema/report.js in the repository if you want the exact reasoning in code-comment form.

Beyond AdSentinel

AuditTool and AdDashboard

AdDashboard is real and running — a public page showing aggregate counts by platform and category from everyone who's clicked Share. AuditTool, a version of the same detection engine meant for platforms or researchers to run against a bulk ad inventory rather than a single page, is on the roadmap but not built yet. If a link to it ever shows up here before it exists, that's a documentation bug — worth reporting the same way as anything else in Module 07.
